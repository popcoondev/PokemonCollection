# クイズモード仕様メモ

## 目的

初代アニメの TV CM 風のテンポで、ポケモンのシルエット当てを楽しめるモードを追加する。

## 基本構成

- クイズ画面は `Aサイド` と `Bサイド` の2段構成
- 起動メニューの `ポケモンクイズ` から入る
- 進行はすべて自動で、タップによる送りは入れない

## レイアウト

- 画面左: 対象ポケモン画像
- 画面右: 名前表示エリア
- Aサイド
  - 左は黒シルエットのみ
  - 右は `？？？？`
- Bサイド
  - 左は正式画像
  - 右は正解のポケモン名

## 進行

- Aサイドを 5 秒表示
- Bサイドを 5 秒表示
- その後、次の問題の Aサイドへ進む
- 秒数は定数化して後から調整できるようにする
- クイズ中は画面タップで起動メニューへ戻れるようにする

## 出題ルール

- 出題対象は `1-1025` の全件
- 出題順はランダム
- 同じポケモンの連続出題は避ける
- 同一セッション内の重複回避までは行わず、直前出題だけを回避する

## 画像方針

- まずは図鑑で使っている既存 PNG を再利用する
- Aサイドのシルエットは、既存 PNG をプログラム側で黒シルエット化できるか検証する
- 既存 PNG からのシルエット生成が不自然な場合のみ、シルエット専用画像の追加を検討する
- 画像は `320x240` の全画面表示を基本にし、その上に一部 UI を重ねる
- ポケモン名は 8 文字程度を基準にし、長い名前は折り返して表示する
- シルエット PoC では、背景透過の崩れ、余白の大きさ、リージョン違いを含む見栄えを確認する

## 音声方針

- クイズモードでは常に BGM を使う
- Aサイド用 BGM と Bサイド用 BGM の 2 曲を使う
- 音源は SD カードに配置する
- 将来的に切替遅延を減らすため、2 曲はクイズモード用に再利用しやすい形で扱う
- Aサイドから Bサイドへ切り替わるときは A を stop して B を再生する
- ループ再生は行わず、A/B の切替ごとに毎回先頭から再生する
- 第1段階の音声形式は `WAV` を前提にする

## 段階導入

### 第1段階

- クイズモードへの遷移
- Aサイド / Bサイドの画面切替
- ダミーでもよいので自動進行の骨組み

### 第2段階

- 全件ランダム出題
- 連続出題回避
- Aサイドのシルエット表示
- Bサイドの正式画像と名前表示

### 第3段階

- BGM 再生
- 画像や音声のキャッシュ方針の調整
- テンポや演出の磨き込み
