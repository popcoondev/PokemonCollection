# 課題メモ

動作確認で見つかった課題をこのファイルに追記して対処する。

## 使い方

- 1課題につき1項目で記録する
- `状態` は `open` / `doing` / `done` を使う
- 再現手順と期待結果を短く残す
- 修正後は `対応メモ` に変更内容を書く

## テンプレート

### 0001. 課題タイトル
- 状態: open
- 発生日: YYYY-MM-DD
- 画面/機能:
- 再現手順:
- 期待結果:
- 実際の結果:
- 対応メモ:

### 0094. BGM ストリーム補充を UI スレッドから分離したい
- 状態: open
- 発生日: 2026-04-26
- 画面/機能: おんがく / たたかう / BGM 再生基盤
- 再現手順: `たたかう` のランダムポケモンモード中に演出を進める、または `おんがく` でディレクトリ移動しながら曲を再生する
- 期待結果: BGM は UI 描画や画面遷移に影響されず継続再生され、タップ操作も鈍化しない
- 実際の結果: 実機では `たたかう` の演出中に BGM がさまざまな箇所で停止し、タップ操作も受け付けにくくなる。`おんがく` でもディレクトリ移動時に再生が一瞬固まる
- 対応メモ: 現状は `serviceMusicStream()` による BGM 補充がメインループ依存で、UI 描画・演出進行・入力処理と同じスレッドで競合している可能性が高い。`SE` と `BGM` のチャンネル分離だけでは不十分で、BGM ストリーム補充自体を専用 worker/task に分離し、`start / stop / next / volume` だけを UI 側から指示する構造へ切り替える。これにより `たたかう` の BGM 停止、`おんがく` の瞬断、入力鈍化の同根問題をまとめて解消する
