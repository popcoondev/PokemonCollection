# 課題メモ

動作確認で見つかった課題をこのファイルに追記して対処する。

## 使い方

- 1課題につき1項目で記録する
- `状態` は `open` / `doing` / `done` を使う
- 再現手順と期待結果を短く残す
- 修正後は `対応メモ` に変更内容を書く

## テンプレート

### 0001. 課題タイトル
- 状態: open
- 発生日: YYYY-MM-DD
- 画面/機能:
- 再現手順:
- 期待結果:
- 実際の結果:
- 対応メモ:

## 課題一覧

### 0001. ボタン押下フィードバックが遅い
- 状態: open
- 発生日: 2026-03-28
- 画面/機能: 全体UI
- 再現手順: ページ送りや検索画面のボタンを押す
- 期待結果: タッチ直後にボタンが即ハイライトされる
- 実際の結果: 同期処理の完了までハイライト表示が遅れる、または次画面表示まで押下中の見た目が残る
- 対応メモ: タッチフィードバックと重い処理を分離する。押下表示は即時反映し、画面遷移や読込は別ステップで進める

### 0002. ページ切り替えのデータ読込が遅い
- 状態: open
- 発生日: 2026-03-28
- 画面/機能: 図鑑詳細のページ送り
- 再現手順: 左右のページ送りボタンでポケモンを切り替える
- 期待結果: 次の図鑑ページができるだけ早く表示される
- 実際の結果: JSON と画像の読込が同期的に走り、次ページの表示まで数秒かかる
- 対応メモ: 文字情報を先に切り替え、画像は非同期または遅延描画で別更新する構成にする

### 0003. 画像読込を非同期化したい
- 状態: open
- 発生日: 2026-03-28
- 画面/機能: 図鑑詳細 / 検索プレビュー
- 再現手順: 画像を伴う画面へ移動する
- 期待結果: 画像以外のUIは先に出て、画像は読込完了後に差し替わる
- 実際の結果: 画像読込完了まで画面更新が待たされる
- 対応メモ: プレースホルダ表示と読込完了後の差し替えが必要

### 0004. 検索機能が重く、仕様も過剰
- 状態: done
- 発生日: 2026-03-28
- 画面/機能: SEARCH画面
- 再現手順: SEARCH画面で番号変更を続ける
- 期待結果: ポケモン番号だけを素早く切り替えられる単純な検索になる
- 実際の結果: アイコンと名称のプレビューのために都度データ参照が発生し、レスポンスが悪い
- 対応メモ: プレビュー画像と名称表示を廃止し、番号だけを切り替える軽量UIに変更した。`ひらく` を押した時だけ詳細を読む

### 0005. ページ送りのタッチ領域を透明オーバーレイ化したい
- 状態: done
- 発生日: 2026-03-28
- 画面/機能: 図鑑詳細のページ送り / 上部バー
- 再現手順: 詳細画面で左右ページ送りやSEARCHを操作する
- 期待結果: 左右ボタンは表示上は完全透明で、画面上端からタブ直上までの範囲内にある幅40px程度の透明タッチ領域として動作する。押した時だけ半透明フィードバックが出る。上部バー全体がSEARCH導線になり、専用のSEARCHボタン表示は不要
- 実際の結果: 左右のページ送りボタンが図鑑情報に重なって見える。今後この透明領域を広げるとSEARCHボタンとも干渉しやすい
- 対応メモ: ページ送りの常時表示アイコンを廃止し、左右端40pxの透明ヒットエリアと押下時オーバーレイに置き換えた。上部バー全体を検索遷移のタップ領域に変更し、SEARCH専用ボタンは廃止した
